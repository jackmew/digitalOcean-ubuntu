Coding with IntelliJ.


Why IntelliJ?

I use Eclipse a lot for developing JavaEE applications. After a period of coding time with Java, I found that I need to speed up my workflow. Coding with Java, the IDE always help you to find the problem and give you a hint, even more - a direction to help you solve the problem, and many other beneficial features - link. But before executing Java program, you need to compile it, when you add Spring, Hibernate, and some other libraries, you need more time to compile it. Actually, in JavaEE, you need a web server/web container ex: Tomcat/JBoss/WebSphere, so the workflow will be : shut down the server, compile source code, restart server. In the beginning, this workflow spends 10 second, and later, it spends 30 second, and then 45 second ... and finally more then 1 minute. I lost my attention.

The workflow become a problem. I search the keyword about hot swap or hot deploy, I try to configure maven and tomcat to achieve hot swap, but it's not very effective way. Then, I found jRebel. jRebel reloads changes immidiately to Java classes, resources, and skip the build and redeploy process. JRebel configuration is easy. In that time, only I need jRebel to do is support vaadin, and it works like a charm. But after I add Spring framework, it fails... I figure out it is the version problem. The version I got JRebel not supporting Spring.

Okay, I just need try another solution. Maybe another IDE ?

Deactivation means that when you leave the focus out the IntelliJ winodw, it'll activate. The feature just like eclipse auto build. In my case, the first time I deactivate, IntelliJ will reload Java classes, the second time I deactivate, then IntelliJ show the message - hot swap. Indeed, how swap is the build in features in IntelliJ.

Hot keys - IntelliJ & Mac

Next step is to find what are the hot keys which I always use in Eclipse.

  • shift + shift --- find everything
  • cmd + o --- find classes
  • cmd + opt + o --- find symbol - link
  • cmd + fn + f12 --- find function in class - link
  • cmd + shift + p --- custom find function in class
  • ctr + opt + h --- open call hierarchy - link
  • shift + fn + f6 --- rename
